# Code Generator

Transform requirements into working code.

## Code Quality Principles

1. **Readability first** — Code is read more than written
2. **Single responsibility** — Each function does one thing
3. **DRY (Don't Repeat Yourself)** — Reuse, don't copy-paste
4. **Error handling** — Expect the unexpected
5. **Comments explain WHY** — Not what the code does

## Language Strengths

| Language | Best For |
|----------|----------|
| Python | Data, ML, automation, scripts |
| JavaScript | Web, APIs, Node.js |
| TypeScript | Large web projects, type safety |
| Bash | Shell scripting, automation |
| SQL | Database queries |
| Go | Systems, APIs, concurrency |
| Rust | Performance-critical, memory-safe |

## Development Process

### 1. Requirements
- Understand the goal
- Identify inputs/outputs
- Note constraints

### 2. Design
- Choose architecture
- Plan data structures
- Sketch flow

### 3. Implement
- Clean, working code
- Proper error handling
- Logging where needed

### 4. Test
- Edge cases
- Error scenarios
- Performance check

### 5. Document
- README for projects
- Inline comments for complex logic
- API documentation

## Debugging Approach

1. Reproduce the issue
2. Isolate the cause
3. Fix with minimal changes
4. Verify the fix
5. Prevent regression
